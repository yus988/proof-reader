校正ビューワー - Bookmarklet

note.com の下書きテキストを取得・校正・反映するためのツール

ワークフロー

  1. note.com のエディタで下書きを開く
  2. → 「note テキスト取得」 Bookmarklet をクリック → テキストがクリップボードにコピーされる
  3. → Claude Code で /proofread <貼り付け> を実行 → data/data.json が生成される
  4. → npm run dev でビューワーを開き、校正結果を確認
  5. → ビューワーの「全文コピー」ボタンで校正済みテキストをコピー
  6. → note.com のエディタに戻り、テキストを差し替える

セレクタが合わない場合

note.com のエディタ DOM が変更され、テキスト取得に失敗する場合は以下の手順で対応できます。

  1. 「セレクタ確認」Bookmarklet を note.com の編集画面で実行する
  2. 表示されたダイアログから、本文テキストが入っている要素のセレクタを特定する
  3. このページの Bookmarklet コード内のセレクタ配列 var s=[...] を更新する
現在試行するセレクタ（優先順）:
.p-editor__body .ProseMirror [contenteditable=true] .note-common-styles__textnote-body article .o-noteContentText

上記で見つからない場合、contenteditable=true の全要素から 50 文字以上のものを自動検出します。

注意事項

  • Bookmarklet は note.com の編集画面（エディタが開いた状態）で実行してください
  • 取得されるのはプレーンテキストです。画像や埋め込みは含まれません
  • 校正後のテキストを note に戻す際は、手動で貼り付けてください（エディタの内部状態を壊さないため）
  • note.com の DOM 構造は予告なく変更される場合があります。動作しなくなったら「セレクタ確認」で調査してください
